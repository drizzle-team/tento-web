# Overview

Build by [drizzle team](https://drizzle.team/)

Tento - is simple yet powerful API for working with Shopify data, including metaobjects, metafields and products.
It also provides a CLI tool for two-way synchronization between your local schema definition and Shopify.
<CodeTab>
<Npm>
@drizzle-team/tento
</Npm>
```typescript
import { tento } from '@drizzle-team/tento';
import * as schema from './schema';

const client = tento({ schema });

await client.metaobjects.designer.list();
```
</CodeTab>

## [Schema](/docs/schema)
Tento lets you declare a definition schema with both **metaobjects** and **metafields**. 
It serves as the source of truth for queries and migrations.
```typescript
import { metaobject } from "@drizzle-team/tento";

export const designer = metaobject({
  name: "Designer",
  type: "designer",
  fieldDefinitions: (f) => ({
    name: f.singleLineTextField(),
  }),
});
```

## [Metaobjects](/docs/metaobject-query)
Tento represents Shopify Metaobject API.
```typescript
await client.metaobjects.designer.list();
```

## [Metafields](/docs/metafield-query)
Tento represents Shopify Metafield API. Now you can declare your schema file with **metafield** definitions and use it for 
metaobject definition reference, products.

## [Products](/docs/product-query)
Tento represents Shopify Product API.
```typescript
await client.products.list();
```

## [Migrations](/docs/migrations)
Tento has a CLI tool for managing Shopify Metaobjects and Metafields schema definitions.
<CodeTab>
<Npx>
tento pull
tento push
</Npx>
```typescript
export default defineConfig({
  schemaPath: './src/schema.ts',
  shop: process.env.SHOPIFY_SHOP_ID!,
  headers: {
    'X-Shopify-Access-Token': process.env.SHOPIFY_ACCESS_TOKEN!,
  },
});
```
</CodeTab>
